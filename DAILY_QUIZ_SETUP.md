# Daily Quiz System - Complete Setup Guide

## 📋 Overview

The Daily Quiz System generates SSC-style quizzes for each subject every day at **12:00 PM IST** (configurable). Users can:
- View their daily quizzes per subject
- Take quizzes with MCQ questions
- Track scores and progress
- Configure preferences (difficulty, time, number of questions)

## 🏗️ Architecture

```
User → /daily-quiz page → Select Subject → /daily-quiz/$subjectId → Take Quiz → /daily-quiz-result
                ↑
        Fetches from daily_quizzes table
                ↑
    Generated by scheduled function at 12 PM
```

## 📊 Database Changes

### New Tables Created:

#### 1. `daily_quizzes`
Stores generated daily quizzes per subject per user
```sql
- id (PK)
- user_id
- subject_id
- quiz_date (YYYY-MM-DD)
- questions (jsonb) - MCQ questions array
- total_questions
- difficulty (easy|medium|hard)
- source (ai|pyq|manual)
- created_at, updated_at
```

#### 2. `daily_quiz_attempts`
Tracks user quiz attempts and scores
```sql
- id (PK)
- user_id
- daily_quiz_id
- subject_id
- quiz_date
- answers (jsonb) - [{question_idx, selected_answer, is_correct}, ...]
- score (correct count)
- percentage
- time_taken (seconds)
- completed (boolean)
- started_at, completed_at
```

#### 3. `daily_quiz_config`
User preferences for daily quizzes
```sql
- id (PK)
- user_id (UNIQUE)
- enabled (boolean, default: true)
- schedule_time (time, default: 12:00:00)
- timezone (default: Asia/Kolkata)
- difficulty (easy|medium|hard|mixed)
- include_pyq (boolean)
- num_questions (5-50, default: 10)
```

### Views Created:
- `today_quiz_status` - Today's quiz status for all subjects
- `daily_quiz_stats_7days` - 7-day performance statistics

Choose based on YOUR hosting platform:

**👉 Full guide: [VERCEL_SETUP_GUIDE.md](VERCEL_SETUP_GUIDE.md)**

#### For Supabase Only (pg_cron)
```bash
# Run this in Supabase SQL editor:
psql "$DATABASE_URL" -f supabase/migrations/20260512_setup_daily_quiz_scheduler.sql

# Prerequisites:
# 1. Enable pg_cron extension in Supabase
# 2. Enable http extension in Supabase
```

#### For GitHub Actions (Alternative)
Create `.github/workflows/daily-quiz.yml`:
```yaml
name: Generate Daily Quizzes
on:
  schedule:
    - cron: '30 6 * * *'  # 12:00 PM IST

jobs:
  generate-quiz:
    runs-on: ubuntu-latest
    steps:
      - name: Trigger Quiz Generation
        run: |
          curl -X POST https://[YOUR-SUPABASE-URL]/functions/v1/generate-daily-quiz \
            -H "Authorization: Bearer ${{ secrets.SUPABASE_KEY }}" \
            -H "Content-Type: application/json"
```

## 🚨 Troubleshooting

### Quiz not generating?
1. ✓ Check if pg_cron job is running: `SELECT * FROM cron.job_run_details`
2. ✓ Verify `LOVABLE_API_KEY` is set
3. ✓ Check if user has subjects with study notes
4. ✓ Manually trigger: `curl -X POST .../functions/v1/generate-daily-quiz`

### No questions showing?
1. ✓ Ensure chunks have `notes` content
2. ✓ Check AI API quota
3. ✓ Verify `daily_quiz_config.enabled = true`

### Scores not saving?
1. ✓ Check Row Level Security (RLS) policies
2. ✓ Verify user authentication
3. ✓ Check browser console for errors

## 🔐 Security Notes

- RLS policies enforce: users can only see their own quizzes
- Service role key needed for scheduled functions
- Quiz generation is user-specific
- Answers stored with user_id
